¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de computadoras interconectadas que utilizan el protocolo de comunicación TCP/IP para compartir información y recursos. Por otro lado, la World Wide Web es un sistema de información que se utiliza en Internet para acceder a documentos en formato hipertexto, denominados páginas web, mediante un navegador web.

¿Cuáles son las partes de un URL?

Protocolo, nombre del dominio, ruta, fragmento.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Los métodos HTTP son verbos que se utilizan en las solicitudes que se envían desde un cliente, como un navegador web, a un servidor web para indicar la acción que se quiere realizar sobre un recurso determinado.

  • GET: Se utiliza para solicitar un recurso específico, como una página web o una imagen, del servidor web. El servidor responde con el recurso solicitado en el cuerpo de la respuesta.
  • HEAD: Es similar a GET, pero solo solicita los encabezados de respuesta, sin solicitar el cuerpo de la respuesta. Este método se utiliza para obtener información sobre un recurso sin descargar su contenido completo, lo que puede ser útil para optimizar el rendimiento de una aplicación.
  • POST: Se utiliza para enviar datos al servidor web para crear o actualizar un recurso específico, como un formulario web o una publicación en redes sociales. Los datos se envían en el cuerpo de la solicitud y el servidor responde con una confirmación o una respuesta de error.
  • PUT: Se utiliza para actualizar o crear un recurso en una ubicación específica en el servidor web. Los datos se envían en el cuerpo de la solicitud y el servidor responde con una confirmación o una respuesta de error.
  • PATCH: Es similar a PUT, pero se utiliza para actualizar parcialmente un recurso en lugar de reemplazarlo por completo. Se envía un conjunto de instrucciones en el cuerpo de la solicitud para actualizar el recurso y el servidor responde con una confirmación o una respuesta de error.
  • DELETE: Se utiliza para eliminar un recurso específico del servidor web. El servidor responde con una confirmación o una respuesta de error.

    • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    El método HTTP que se debe utilizar al enviar un formulario HTML es POST. El motivo es que el método POST envía los datos del formulario en el cuerpo de la solicitud HTTP

    ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    Se utiliza GET. El navegador web envía una solicitud GET al servidor web correspondiente al URL especificado para obtener el contenido de la página web solicitada. El servidor web responde a la solicitud GET con la página web solicitada en el cuerpo de la respuesta HTTP

    Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    Una respuesta HTTP con código 200 significa que la solicitud HTTP se ha procesado correctamente y que se ha enviado una respuesta satisfactoria al cliente. En otras palabras, el código 200 indica que no ha ocurrido ningún error y que la solicitud ha sido exitosa.

    ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404. El error 404 significa que el servidor web no pudo encontrar el recurso solicitado por el usuario, y esto puede ser causado por una variedad de problemas, como un enlace roto o una página eliminada.
    Como desarrollador web, es importante garantizar que el sitio web esté en funcionamiento y que los usuarios puedan acceder a todos los recursos del sitio sin problemas. Si un usuario informa un error 404, el desarrollador debe investigar y solucionar el problema lo antes posible para garantizar que los usuarios puedan acceder al recurso solicitado.

    ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500. El error 500 es un error interno del servidor que indica que se ha producido un error en el servidor al procesar la solicitud del cliente.

    ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Cuando un atributo HTML5 está deprecado o desaprobado (deprecated), significa que se considera obsoleto y que ya no se recomienda su uso en el desarrollo de sitios web. Esto se debe a que se han introducido nuevas características en HTML5 que reemplazan la funcionalidad del atributo desaprobado o porque su uso puede generar problemas de compatibilidad con navegadores más modernos. Algunos ejemplos son: center, align, font, y frameborder.

    ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    HTML5 presenta varias mejoras significativas en comparación con HTML 4, incluyendo nuevos elementos, mejoras en multimedia y formularios, mayor compatibilidad con dispositivos móviles, menos dependencia de complementos y mejoras en la semántica.

    ¿Qué componentes de estructura y estilo tiene una tabla?

    Una tabla HTML consta de filas, celdas, encabezados de columna y fila, y se pueden aplicar varios atributos de estilo para personalizar su apariencia.

    ¿Cuáles son los principales controles de una forma HTML5?

    Algunos de los más comunes son input de texto, input de selección, input de archivo, botones, input de número, etc.

    ¿Cuál es el ciclo de vida de los sistemas de información?

    El ciclo de vida de un sistema de información (SLC, por sus siglas en inglés) es un proceso que describe las diferentes etapas por las que pasa un sistema de información, desde la concepción inicial hasta su eliminación.

    ¿Cuál es el ciclo de desarrollo de sistemas de información?

    El ciclo de vida de los sistemas de información (SDLC, por sus siglas en inglés) es un enfoque estructurado para el desarrollo de sistemas de información que consta de una serie de fases interdependientes y secuenciales.
    Fases: planificación, análisis, diseño, implementación, pruebas, mantenimiento.
